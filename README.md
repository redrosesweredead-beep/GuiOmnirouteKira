 # GuiOmnirouteKira: Графический интерфейс для интеграции Claude LLM через прокси-сервер OmniRoute

### Аннотация
В данном репозитории представлено программное решение для оптимизации взаимодействия с языковой моделью Claude Sonnet 4.5. Проект реализует асинхронную графическую оболочку, решающую задачи автоматизации управления прокси-средой, корректной обработки событий ввода и экспорта структурированных данных.

### 1. Архитектура и технический стек
Программный комплекс разработан на языке Python и базируется на следующих компонентах:
* **Интерфейс**: Библиотека `customtkinter` (объектно-ориентированный подход к построению GUI).
* **Сетевой протокол**: HTTP-запросы через библиотеку `requests` к локальному адресу `127.0.0.1:20128`.
* **Прокси-сервер**: Интеграция с OmniRoute для маршрутизации трафика к API Claude.
* **Управление процессами**: Модуль `subprocess` для автоматической очистки стека процессов `node.exe` перед инициализацией.

### 2. Технические особенности реализации
#### 2.1. Нормализация событий прокрутки
Реализован алгоритм сквозной обработки событий `<MouseWheel>`. В отличие от стандартных реализаций `tkinter`, события ввода перехватываются на уровне дочерних виджетов (текстовых полей с кодом) и делегируются родительскому холсту, что обеспечивает непрерывную вертикальную навигацию.

#### 2.2. Обработка и экспорт программного кода
Интерфейс оснащен парсером Markdown-разметки, который:
* Изолирует программный код в специализированные контейнеры.
* Сопоставляет синтаксические теги с соответствующими расширениями файлов.
* Обеспечивает прямой вывод данных (I/O) в локальную файловую систему с кодировкой UTF-8.

#### 2.3. Визуальная оптимизация
Графическая оболочка выполнена в темной цветовой палитре (основной цвет: `#0e0e10`). Дизайн минимизирует когнитивную нагрузку при длительной работе и соответствует современным стандартам визуализации интерфейсов ИИ.
